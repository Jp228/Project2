    S - Single-responsibility Principle

  • A class has only one and only one job. It was used in the folder calculations in addition.py, subtraction.py, multiplication.py and division.py.

      • O - Open-closed Principle

  • A class that does not change the method each time it runs. This was used in the calculations in addition.py, subtraction.py, multiplication.py and division.py.

      • L - Liskov Substitution Principle

  • States that the child class should be suitable to its parent class or its super class.

      • I - Interface Segregation Principle

  • The principle states that the code should not be forced to implement any methods that it does not need and does not use.

      • D - Dependency Inversion Principle

  • This principle states that high level modules should not depend on low level modules instead they should use abstraction. This is used in the program in calculation.py where we used the abstract method.
